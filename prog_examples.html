General Programming Concepts & Data Structures

Humans Vs Zombies

Humans Vs Zombies is a simulation based on Conway's Game of Life. Zombies hunt humans to infect them and Humans flee to survive.

It uses a polymorphic algorithm to calculate different logic patterns. Zombies and Humans are objects of the "Organism" class and share core functionality. Humans can repopulate and Zombies can move diagonally, infect, move before Humans, prioritize attacking, and perish if they move 3+ times without eating.

This taught me about how to use object pointers in C++ and to utilize a dynamic array to manage all entities. The game window is being displayed inside of a "Gnome Terminal".

Maze Solver

A console application that reads a text maze and utilizes a pathing algorithm to navigate through the maze and find the exit and display the solution to the maze.

The pointer moves through the maze and looks for dead-ends. When it finds a dead-end, it backtracks and marks that path as a dead-end then continues searching the next path.

This program taught me how use a stack data structure to create a pathing algorithm. I was given starter code to study how a stack works and then utilize it.

Object Oriented Programming & Polymorphism

Shapes Screensaver

This is a low level project from my first course in Java and also my introduction to Object Oriented Programming. This screensaver program uses a polymorphic Shape object to share movement and collision methods.

On every collision with a wall, shapes flip direction and each shape has a chance to change state in many ways. They all may toggle outline/solid fill or change colour. Squares and Circles also have a chance of morphing into an instance of the others' class so a Circle has a chance to upcast to Shape and then downcast to a Square object and keep all other existing state settings.

On each click of the mouse, a shape is added to the program at runtime.

Airplane Seating

This program uses a 2D array to represent 16 seats of an airplane seating arrangement. The top 8 are 1st class and the bottom 8 are coach.

The User will input their name and then select if they want to sit in 1st class. After that they will select whether they prefer an aisle seat. Using this information, the program will check the 4 seats that apply to any combonation of selections and if the first seat is taken it will then check the next. At the end of seating, the program outputs a report showing all passengers names with their seat number.

This project was partially to learn about objects and storing objects to an array, the other learning component involved using a GUI for the first time.
